# ЛР8: зона пешей доставки еды в Калуге

Небольшое локальное веб-приложение для лабораторной работы по географическим API.

Вариант индивидуального задания: `1`  
Город по варианту: `11 — Калуга`

## Что делает приложение

Приложение определяет зону, в которой пеший курьер сможет доставить еду из ресторана за указанное пользователем время.

Решение построено так, чтобы на защите явно показать все обязательные пункты:

1. `Геокодинг` — поиск ресторана и клиента по адресу через `Nominatim API`.
2. `Тайлинг` — отображение карты через `2GIS RasterJS API`.
3. `Определение геохеша территории` — локальный REST endpoint `/api/geohash`, который вычисляет geohash и ячейку территории для выбранного ресторана по стандарту `CTA-5009`.
4. `Построение маршрутов` — расчёт пешеходного маршрута через `Valhalla Route API`.
5. `Построение изохрон` — расчёт зоны доступности пешком через `Valhalla Isochrone API`.

## Использованные API и источники

1. `Nominatim Search API`  
   Документация: https://nominatim.org/release-docs/latest/api/Search/
2. `2GIS RasterJS API`  
   Документация: https://docs.2gis.com/maps/others/rasterjs/overview
3. `Valhalla Route API`  
   Документация: https://valhalla.github.io/valhalla/api/turn-by-turn/api-reference/
4. `Valhalla Isochrone API`  
   Документация: https://valhalla.github.io/valhalla/api/isochrone/api-reference/
5. `Valhalla public demo server`  
   Описание demo-сервера: https://valhalla.github.io/valhalla/
6. `CTA-5009 Geohash specification`  
   Спецификация geohash: https://cdn.cta.tech/cta/media/media/resources/standards/cta-5009-a-final.pdf
## Структура проекта

- `server.js` — локальный HTTP-сервер и единый API-слой приложения.
- `index.html` — интерфейс для демонстрации лабораторной.
- `styles.css` — оформление.
- `app.js` — клиентская логика, работа с картой и вызовами API.
- `package.json` — запуск проекта.
- `smoke-test.js` — автоматическая проверка ключевых сценариев.

## Как запустить

Нужен только `Node.js 18+`.

```bash
npm start
```

После запуска открыть:

```text
http://localhost:3000
```

Проверка ключевых сценариев:

```bash
npm run smoke
```

## Как показывать лабораторную

### 1. Геокодинг

В поле ресторана или клиента введите, например:

- `улица Кирова, 1, Калуга`
- `улица Ленина, 50, Калуга`
- `abracadabra kaluga` — для демонстрации ошибки

После поиска приложение покажет найденные объекты и позволит выбрать нужный.

### 2. Тайлинговая карта

На карте используется `2GIS RasterJS API` с ключом доступа к картографическим тайлам 2GIS.

### 3. Геохеш территории

После запуска анализа приложение:

- вычисляет geohash точки ресторана;
- рисует прямоугольник соответствующей geohash-ячейки;
- выводит код geohash и координаты центра ячейки.

### 4. Маршрут

Если задан клиент, приложение строит пешеходный маршрут и показывает:

- длину пути в км;
- время в минутах;
- вывод: возможна ли доставка за заданное время.

### 5. Изохрона

Для ресторана строится изохрона пешей доступности за введённое число минут.

Дополнительно можно нажать `Скопировать GeoJSON изохроны` и вставить объект в любой онлайн-сервис для показа GeoJSON.

## Пять демонстрационных примеров

### Пример 1. Успешная доставка

- Ресторан: `ул. Кирова, 1`
- Клиент: `центр Калуги`
- Лимит: `20 минут`
- Ожидаемый результат: доставка возможна

### Пример 2. Недостаточно времени

- Ресторан: `ул. Кирова, 1`
- Клиент: `ул. Ленина, 50`
- Лимит: `15 минут`
- Ожидаемый результат: доставка невозможна

### Пример 3. Увеличение лимита времени

- Ресторан: `ул. Кирова, 1`
- Клиент: `ул. Ленина, 50`
- Лимит: `30 минут`
- Ожидаемый результат: доставка возможна

### Пример 4. Только зона доступности

- Ресторан задан
- Клиент не задан
- Лимит: `12 минут`
- Ожидаемый результат: строится только изохрона без маршрута

### Пример 5. Ошибка и нестандартная ситуация

- Поиск `abracadabra kaluga`
- Ожидаемый результат: приложение корректно сообщает, что в пределах Калуги ничего не найдено

Дополнительная нестандартная ситуация:

- ресторан и клиент совпадают;
- приложение показывает почти нулевой маршрут и корректно отрабатывает случай.
